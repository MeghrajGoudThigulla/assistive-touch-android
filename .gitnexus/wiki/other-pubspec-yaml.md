# Module: pubspec.yaml

## Overview

The `pubspec.yaml` file is the central configuration file for the `assistive_touch_android` Flutter project. It defines the project's metadata, sets the required Dart SDK version, and manages all external package dependencies and Flutter-specific configurations like assets and fonts.

This document serves as a guide to understanding the project's setup and dependencies as declared in this file.

## Project Metadata

This section defines the identity of the application.

-   **`name`**: `assistive_touch_android`
    -   The official name of the package.
-   **`description`**: `"A new Flutter project."`
    -   A brief, one-line description of the project's purpose.
-   **`publish_to`**: `'none'`
    -   This setting prevents the package from being accidentally published to [pub.dev](https://pub.dev), indicating it is a private application rather than a reusable library.
-   **`version`**: `1.0.0+1`
    -   The application's version, following the `major.minor.patch+build` format.
        -   `1.0.0` is the user-facing version name (`versionName` in Android, `CFBundleShortVersionString` in iOS).
        -   `+1` is the internal build number (`versionCode` in Android, `CFBundleVersion` in iOS).

## Environment

```yaml
environment:
  sdk: ^3.9.2
```

This section specifies the Dart SDK version constraints for the project. The caret `^` syntax indicates that the project is compatible with any Dart SDK version from `3.9.2` up to, but not including, `4.0.0`. This ensures that the project is built with a stable and compatible version of the Dart language and tools.
